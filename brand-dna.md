# Brand DNA — 李子礽的互联网法律作品集视觉 DNA

> 这是本作品集的底层视觉与表达规范。后续页面、报告、组件和简历补充材料都应优先遵守这里的判断。

---

## 1. 核心定位

这个网站不是普通学生作品集，也不是游戏同人站。它要表达的是：

- 李子礽（Finley）是申请游戏法、互联网合规方向的实习律师候选人。
- 基础来自两年公司法务实务、法考 A 证、合同审查、隐私合规、诉讼支持与内部培训。
- 研究方向聚焦互联网法律问题：虚拟财产、AIGC 权属与侵权处理、数据合规、UGC 平台责任、避风港规则与 PIPL 删除权。
- 个人语境来自重度游戏玩家身份，但对外表达应专业、克制、有判断。

一句话方向：**用玩家文化的视觉语言，呈现一个能处理互联网法律问题的实务型候选人。**

---

## 2. IP 固定三色

全站主视觉固定为绿、粉、黑。其他颜色可以作为项目看板或小标签的局部点缀，但不能改变主基调。

| 色名 | 色值 | 用途 |
|------|------|------|
| 荧光绿 | `#A7F432` | 主识别色、姓名阴影、墨迹底色、重点标记 |
| 粉色 | `#EF4B7A` | 第二识别色、阴影、强调线、局部标签 |
| 墨黑 | `#171512` | 正文、边框、主标题、卡片投影 |
| 暖纸色 | `#FFFAF0` | 卡片底色、报告阅读背景 |
| 底色米灰 | `#F4EFE4` | 页面底色 |

比例原则：

- 黑色是骨架，负责专业感和法律感。
- 绿粉是识别，不是整页铺满的装饰。
- 姓名和主视觉可以用绿粉阴影，但饱和度与透明度必须控制，不能糊眼。
- 项目卡片可以保留少量橙、青、黄等颜色作为分类条，但全站背景和姓名主视觉不混搭黄蓝。

## 3. Avatar / IP 形象规范

头像文件：`assets/avatar.png`

形象关键词：

- 黑色短发与厚刘海。
- 尖耳与玩家感轮廓。
- 锐利眼神、轻微自信笑容、浅腮红。
- 紫色西装、白衬衫、红色格纹领带。
- 气质是「游戏玩家 + 实务法律人」，不是可爱吉祥物，也不是严肃证件照。

使用规则：

- 头像可作为个人 IP 识别，不使用个人照片。
- 对外不写 Nintendo / Splatoon 官方关联，不使用官方 Logo、角色名、截图或任何可能造成授权误认的素材。
- 文案表述用「个人自捏 avatar」「玩家文化视觉参考」即可。
- 头像适合放在姓名区旁边作为小型身份贴纸，或放在 About、Contact、品牌页；不要塞进已经复杂的泼墨主视觉里。
- 头像旁可以使用 `FINLEY` 小型 ID 标签，作为英文名识别，不再额外解释。
- 不拉伸、不套过重滤镜、不和绿粉墨迹互相抢主体。

## 4. 字体基因

已接入本地字体：

| 场景 | 字体 | 文件 |
|------|------|------|
| 中文标题首选 | 创客贴金刚体 | `assets/fonts/chuangkit-jingang.otf` |
| 中文卡片标题 / 能力标题 | Source Han Sans CN Bold | `assets/fonts/source-han-sans-cn-bold.otf` |
| 中文正文首选 | Source Han Sans CN Regular | `assets/fonts/source-han-sans-cn-regular.otf` |
| 英文小标签 | Avenir Black | `assets/fonts/avenir-black.ttf` |
| 姓名字标源字体 | 上图东观体粗体 | `assets/fonts/shangtu-dongguan-bold.ttf` |
| 姓名字标 | 上图东观体粗体导出的加粗 SVG 路径字标 | `assets/name-wordmark.svg` |

CSS 使用：

```css
--display: "Chuangkit Jingang", "Source Han Sans CN Local", "PingFang SC", "Microsoft YaHei", sans-serif;
--heading-sans: "Source Han Sans CN Bold Local", "Source Han Sans CN Local", "PingFang SC", "Microsoft YaHei", sans-serif;
--sans: "Source Han Sans CN Local", ui-sans-serif, system-ui, -apple-system, "PingFang SC", "Microsoft YaHei", sans-serif;
--label: "Avenir Black Local", "Source Han Sans CN Bold Local", ui-sans-serif, system-ui, sans-serif;
--mono: ui-monospace, "SFMono-Regular", "Cascadia Code", "Roboto Mono", "Menlo", "Consolas", monospace;
```

使用原则：

- H1、H2 和需要强识别的展示标题使用金刚体。
- 首页姓名使用 `assets/name-wordmark.svg` 路径字标；黑色主体用同色描边加粗，绿粉衬底直接做在 SVG 内，避免 CSS 多层投影叠脏。
- 卡片标题、能力点标题、项目标题使用更粗的黑体，不使用金刚体，避免页面到处都像海报。
- `Experience Docket`、`Research Loadout` 等英文小标签使用 Avenir Black。
- 正文、报告正文、说明文字优先使用思源黑体，保证阅读稳定。
- 大标题可以很大，但行距要紧，不能让页面显得空或者像海报硬凑。
- 正文不追求花哨，重点是清楚、可信、像真实法律作品。
- 字体公开部署前确认金刚体授权边界；思源黑体可作为更稳的开源正文基础。

## 5. 版式语言

关键词：案卷、证据、看板、报告、墨迹、玩家语境。

页面应像一个有设计感的候选人档案，而不是营销落地页。

- 首页负责快速建立身份：实习律师候选人、公司法务基础、互联网法律研究、玩家语境。
- Finley 作为英文名只放在头像 ID 小标签里，避免全站重复刷屏。
- 轻赛博感来自黑底信息条、细线、编号、扫描感背景和绿粉局部发光；不要做满屏紫蓝霓虹。
- JD Fit 区块只放和岗位最相关的信息：游戏、AI、数据合规。
- 可迁移能力区块要讲清楚「实务基础如何迁移」，不要写成辩解。
- 项目经历区块用案卷卡片呈现，标题要像真实工作项，描述要具体。
- 研究报告区块要像可阅读的法律成果，不要像博客列表。
- 履历区的研究类卡片必须对应已有五份报告原题，不把研究方向扩写成未提供材料支撑的“项目经历”。
- 联系区块应简洁，不要过度煽情。

卡片规则：

- 卡片半径保持 8px 左右。
- 可以用粗边框和黑色投影，但投影不要把内容压脏。
- 不做卡片套卡片。
- 小标签用于信息分类，不用于解释页面怎么用。

## 6. 文案声线

整体声线：清楚、直接、有判断，不自我贬低，不用 AI 味总结。

优先写法：

- 「两年的法务经验打下了夯实基础及工作能力。」
- 「研究方向包括虚拟财产、AIGC 权属与侵权处理、数据合规、UGC 平台责任、避风港规则与 PIPL 删除权。」
- 「我习惯先还原业务流程，再识别合同、数据、平台规则和责任边界中的法律风险。」

避免写法：

- 「这部分证明我为什么能进入游戏法团队。」
- 「我没有互联网公司法务经历，但是……」
- 「本作品集将岗位要求拆成三类证据。」
- 「赋能、闭环、底层逻辑、抓手、方法论」等空泛表达。

可以使用第一人称，但要像求职材料里的自述，不像社媒小作文。

## 7. 报告与 Markdown 呈现

Markdown 报告要保留结构感，但不能照搬 Obsidian 的花色。

- H1/H2/H3 使用黑色或站点绿粉体系中的低饱和强调色。
- 报告页标题优先服务阅读，不做首页式海报大字；字号、行距和宽度要让标题自然分行。
- 报告正文一级标题不得使用首页式巨型字号；应明显强于正文但保持阅读节奏，避免压迫正文区域。
- 主标题不要塞过多括号解释。括号中的制度别名、说明性内容应下放到导语或正文首段。
- 必要时允许手动分行，例如主制度名一行、研究类型一行，避免单字或短词被挤到孤立一行。
- Markdown 原文中的加粗必须完整保留；调整标题颜色或组件样式时，不得删掉正文、表格、列表和 callout 标题里的强调。
- Obsidian 内部引用残留不能裸露在网页里，例如「详见 XXX」「[[XXX]]」。没有真实网页链接时直接删除或改写进正文。
- Obsidian 图片附件不能一概删除；能在附件目录找到原图时，应复制进站点资源并按报告图片组件嵌入。
- 如果图片内容本质是表格，优先解析成网页原生表格；原图仅在无法可靠转写时保留。
- Callout 可以保留，但要转成网站组件，避免默认引用块样式。
- 表格必须可横向滚动，边框清楚，不能挤成截图感。
- 报告正文段落使用 2em 首行缩进；表格、列表、标题、callout 不强制缩进。
- 报告阅读页应以正文阅读为主，装饰降噪。
- 目录、标签、返回按钮都要服务阅读，不做过多视觉噪音。

## 8. 禁忌清单

| 类型 | 禁止 |
|------|------|
| 配色 | 全站黄蓝、绿粉黄蓝混搭、过重绿粉阴影、冷灰蓝 AI 模板感 |
| 文案 | 解释自己为什么不够匹配、空泛能力词堆叠、第三人称模板腔 |
| 图片 | 官方游戏 Logo、官方截图、让人误以为有授权的 IP 素材 |
| 布局 | 过大留白、每块都像宣传海报、卡片套卡片 |
| 字体 | 标题回退成普通系统宋体、正文使用难读装饰字体 |
| 动效 | 大幅漂浮、无意义循环动画、影响阅读的滚动效果 |

## 9. 自检问题

每次改完页面后问三件事：

1. 面试官能不能在 10 秒内看懂我是谁、要投什么方向？
2. 这块内容是在增加可信度，还是只是在解释设计？
3. 这张截图像真实候选人的作品集，还是像套模板生成的网站？
